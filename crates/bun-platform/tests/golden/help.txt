bun-platform is the standalone execution boundary for the bun-dev skill. It audits Bun-first repos, plans and applies bounded safe fixes, validates repo state, benchmarks the hot path, and maintains vendor-backed bun-dev references.

Usage: bun-platform [COMMAND]

Commands:
  audit             Audit a repository for Bun platform findings.
  list-rules        List bun-dev rule identifiers.
  explain           Print the full markdown for one bun-dev rule.
  plan-fixes        Plan deterministic safe fixes without mutating the repo.
  apply-safe-fixes  Apply deterministic safe fixes.
  validate          Audit the repo and then run validation commands.
  benchmark         Measure hot-path performance for audit and fix planning.
  release-sync      Maintain vendor-backed bun-dev references and indexes.
  doctor            Inspect bun-platform installation, state paths, and skill integrity.
  completions       Generate shell completions.
  help              Print this message or the help of the given subcommand(s)

Options:
  -h, --help
          Print help (see a summary with '-h')

  -V, --version
          Print version

Examples:
  bun-platform audit --root .
  bun-platform plan-fixes --root .
  bun-platform benchmark --root ./fixtures/github-actions --format json
  bun-platform release-sync --status
  bun-platform release-sync --dry-run
