# Expo Native Lane

Use this lane when Expo, EAS, or React Native concerns are primary.

## Priorities

1. Preserve native UX and release safety.
2. Prefer Expo-managed workflows over custom native divergence unless the repo clearly requires otherwise.
3. Validate with Expo/EAS/native checks already defined by the repo.

## Escalate To

- `$ui-audit` for UX and accessibility review
- `$convex-audit` when backend contract work is the real bottleneck
